What does this offer ?

This API and context offer a new way of seeing how we write our frontend applications.

In fact, Here is an:

Application-wide Type-safe lazy-loaded state manipulation

It allows:

  • Enumerate all states in the application at dev-time
  • Powerful auto-completion for all states (no selectors, no string)
  • Subscribe and manipulate all states, from anywhere
  • Typed all the way up-and-down

How will you use it​

In a nutshell, here is a small example:

application.js
import { api, createApplication } from "react-async-states"

let Application = {
  auth: {
    login: api<Pricipal, Error, never, [LoginRequest]>()
  },
  users: {
    list: api<Page<User>, Error, "Timeout", [string]>(),
  }
}

export let app = createApplication<typeof Application>(Application)

UsersList.js
import { ProducerProps } from "react-async-states"

function fetchUsersProducer(props: ProducerProps<Page<User>, Error, "Timeout", [string]>) {
  let controller = new AbortController()
  props.onAbort(() => controller.abort())
  
  let [query] = props.args;
  return searchUsers(query, controller.signal)
}

export function UsersListComponent() {
  let {state, run} = app.users.list.use()
  // run is called with a single string argument
  return <UiAround state run />
}

What's the idea behind it​

This abstraction does an analogy to resources and APIs from the backend land.

In practice, we make resources that exposes APIs to be used by our clients.

So, this abstraction does a type mapping over a two-level deep object where you define in advance the types of the states in your application.

This abstraction doesn't force you to define the whole producers of any state in your application, you should provide only types. And this abstraction gives you a similar object, but rather than your API configuration, you get a token with this type:


export type Token<T, E, R, A extends unknown[]> = {
  (): Source<T, E, R, A>,
  inject(
    fn: Producer<T, E, R, A>,
    config?: ProducerConfig<T, E, R, A>
  ): Token<T, E, R, A>
  use<S = State<T, E, R, A>>(
    config?: UseConfig<T, E, R, A, S>,
    deps?: any[]
  ): UseAsyncState<T, E, R, A, S>,
}
  • (): Invoking the token returns the Source object created for this state. If not created, it will throw an error that you should obviously catch in dev mode, it even includes the full path of the API and the function to be called
  • inject(producer, producerConfig): This effectively creates your state. If it already was created, the producer and the config will be patched.
  • use: This is useAsyncState under the hood, it accepts the properties related to subscription from useAsyncState.
